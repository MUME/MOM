FAQ (Frequently Asked Questions)

  • Supported Operating System(OS):
    This program works with Microsoft Windows 98/98SE/ME/NT/2000/XP
  • Minimum Computer Requirements:
    Atleast 32MB system memory.
    Atleast 50MB free disk space.
    CPU 300MHz or above. More memory helps
  • Supported and tested Clients:
    - JMC (MOM is optimized for JMC)
    - PowTTY(should work, if problems inform me ASAP)
    - WinTin (should work)
    - ZMUD (some say it works, others say it doesn't, test and let me know)
    - Windows Telnet (doesn't work with some Windows XP, I'm looking into this)
    - Other clients. Test them and let me know
  • MOM works only and only on the installed computer, with only YOUR map!
  • Backup your "map.txt", because if you lose it, then it is gone!
  • MOM comes with some basic mapped areas, cities and widely used roads
  • To connect to MUME, read the manual.

    Q: Wrong protocol or connection state for the requested transaction or request(40006)
    A: MOM is running, but your client(JMC/Telnet/..) is not running or not connected to MOM,
    this error doesn't cause any bad. You can still modify your map offline.
    You should delete your -log.txt- file sometimes, because it can get too large.

    Q: My map does not locate
    A: The character you play must have MUME command CHANGE COLOUR LOOK with correct colour!
    The MUME colour must be one of the colours listed in Options/Change colour look/-colour-
    There is a difference between -bold green- and -green-!

    Q: My map located(synced) properly, but for some reason it won't follow
    A: MOM only accepts following movement commands:
    [n e s w u d] or their combinations [nesewudwn], [n;e;s;e;w;u;d;w;n],
    MOM does not support movement typed like [eas sou so nort up dow],
    that command is not recognized by MOM and sent straight to MUME.

    Q: Gathering room data: GETTING DATA FROM MUME FAILED!

    mapping to closed direction or,
    mapping too fast, - you cannot do anything else when the MOM reads room data
    there was some/narrate/yell spam in room, that sometimes comes between
    descripton and exits data and can ruin the format. Then you will either see,
    Q: Black square border appears around the room
    A: This means the data is invalid or the mapper could not gather EXITS information,
    in some cases entering the room shows the exits, but the separate command will not.
    Then just manually click all the existing exit directions True.

    Q: Is there any fast command for MAPMODE (or some similar)
    A: Yes. _help gives overall command list, and the command you are looking for is _tool
